它管哪一类事

不做，也没人会来追究你的事。

不是「忘了开会」那种——开会有人找你，忘了有后果。它管的是另一类：

  • 椅子上的衣服堆了三天，没有后果逼你收，所以你不会收。
  • 想种草某个东西，念头一闪而过，几天后再也想不起。
  • 洗碗、回消息：每件都不难，就是启动不了。
  • 同一个提醒喊到第三次，对你已经彻底失效了。

02
设计立场

必须做的事，别押在大模型上。

任何由大模型驱动的 Todo List 都是不稳定的——这是刻意的边界。

  • 它会漏掉一条、会误判优先级、会在你最需要它的时候压错判断。
  • 把「不做有后果」的事交给它，风险太高。
  • 那些事留在你本来就在用的确定性工具里。
  • Traceflow 只补 Todo List 的盲区，不抢它的活。

它具体做什么

Todo List 做不到的，它来。

  • 每天换个说法、结合你当下的状态提醒习惯，防止提醒失效。
  • 快速梳理几件家务，带你一件件过，而不是甩一张清单。
  • 你种草了 xx，百无聊赖时它主动提，还用模型知识 + 联网搜索给你指路。
  • 一句话记下的念头，由它代收、整理，之后提醒你回顾。
